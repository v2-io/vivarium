# What vivarium is for

Vivarium is both a place to play in a living constructed world and a laboratory that can host *in vivia* results — those purposes coexist, and claim discipline exists so the laboratory half does not dissolve into an unprincipled game, nor the fun half into endless rewrites.

## Formal Expression

1. **Play.** The project is allowed to pursue open-ended enjoyment of world texture, exploration, and sim-game craft (RimWorld / Dwarf Fortress lineage). Fun is a real objective, not a pretext.
2. **Laboratory.** The project authors worlds in which ASF/AAT quantities can be known by construction — law $\theta$, state $\Omega$, chance $\varepsilon$, compute-shortfall — so results can be cited *in vivia* ( #def-in-vivia): richer dynamics than a toy model, more legibility than field data.
3. **Shared walls.** Both purposes share the same non-negotiable engineering walls ( #post-determinism-as-ontology, #form-core-view-wall, #post-represent-by-consequence). Those walls exist so play does not force a later teardown and so laboratory claims remain honest.
4. **Phase ordering of work.** While claim segments are incomplete, **claim truth outranks demo polish**. Play that does not mint unowned claims is welcome; play that requires a new physical or architectural assertion requires the segment first (or an explicit provisional mark).

## Discussion

Play and laboratory share coherence as the load-bearing property: a world that is fun because it is internally consistent is the same consistency *in vivia* citation needs. The failure mode is treating "fun now" as license to leave law and claims implicit in code and chat.
