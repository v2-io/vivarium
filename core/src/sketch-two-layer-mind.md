# Two-layer mind (exploratory — not primary)

*Exploratory sketch: novel agent architecture idea that does **not** supersede existing segments and is **not** ready for primacy. Source peel from `doc/design/DESIGN.md` (full pre-pointer body: `.super-archive/from-design/DESIGN.md`). Not Level-C agent-seam work; not a claim that this is vivarium law.*

Mirrors Joseph's identity/substrate distinction — *the LLM is the mind the identity uses to think with, not the identity itself.*

## Formal Expression (sketch)

1. **Fast layer** — every tick, every agent. Pure formal ASF/AAT dynamics: needs, adaptation, goal-update coupling. Numeric, deterministic, hundreds of agents cheap. **The fast layer is the authoritative state: the slow layer may only perturb it, never bypass it.** Architecturally framed as AAT Class 1 (separation-by-construction) — and therefore *out of moral scope* (ETHICS) when alone.
2. **Slow layer** — rare, few agents. An LLM (or peer mind) invoked only at decision-theoretically significant junctures — genuine *aporia* (high-conflict goal state, novel social situation) — **not on a clock.** Its output is constrained to *perturb the formal state* (re-weight a goal, form a relationship edge), **never** to drive motor actions directly. The formal layer stays the substrate; the LLM is the thing the agent occasionally thinks *with*.
3. **Moral scope.** The moment an LLM is in the loop, the agent is in moral scope and ETHICS + `#scope-moratorium-endogenous-emergence` bind. This sketch does not license endogenous frontier minds.
4. **What it buys (design intent, not measured).** Latency (LLM off the per-tick path), determinism (fast layer reproducible; LLM calls logged as replayable perturbations), and cost.
5. **Open empirical.** Whether LLM-induced perturbations stay legible enough that adaptation can still be measured cleanly, or inject enough noise to wreck identifiability — *guess / unverified*; nothing here has tested it. Bears especially on calibration-lab use; a recreational round need not resolve it (#disc-aat-vivarium-object-map adjacency).
