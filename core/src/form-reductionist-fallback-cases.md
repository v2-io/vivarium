# Three reasons to leave pure reductionism (and they need different machinery)

Preferred default: fine-grained reductionist simulation with macro behaviour emergent. When that fails, the *reason* matters — three cases demand different machinery. Conflating them hides which mechanism an aspect needs.

## Formal Expression

1. **Default.** Prefer fine-grained reduction with emergent macro, within represent-by-consequence and fidelity budgets ( #post-represent-by-consequence , #form-fidelity-invariant ).
2. **Case A — laws unknown.** No trustworthy micro-model (ecosystems, consciousness-adjacent). **Impose** a phenomenological macro and declare that it is imposed.
3. **Case B — laws known, intractable at fine grain.** Physics exists; cost forbids running it everywhere (continental erosion, fluids). Run fine **only** in the observed/active locus; cheap surrogate elsewhere. *Most of vivarium.*
4. **Case C — laws known and affordable, not yet validated** to integrate to the right macro. Need the reconciliation harness / regime probes ( #norm-regime-probes ) **before** trusting emergence. Failures cluster here: the model runs, looks plausible, and silently fails to reproduce the macro it replaced.
5. **Do not conflate.** A phenomenological macro (A), a cost surrogate (B), and an unvalidated fine model (C) are different debts. Case C is not cured by more detail alone.
