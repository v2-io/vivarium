# vivarium

*An enclosed living world kept for observation.*

A 2D simulation game in the lineage of RimWorld (storytelling colony sim) and
Dwarf Fortress (deep emergent world dynamics), with graphics in the direction of
top-down action-RPGs (Zelda-like). Its distinguishing bet is **simulation-grade
agents and community dynamics** built on the Agentic Systems Framework (ASF /
AAT) rather than the cheap utility-AI that colony sims normally use.

The name is deliberate: a *vivarium* is an enclosed space for keeping living
things **under observation**. This project is two things wearing one skin —

1. a game world, and
2. a high-identifiability environment in which ASF/AAT agent dynamics can be
   *measured*, because we control the dynamics and can log every state.

That duality is the point, not an accident. See [`DESIGN.md`](DESIGN.md).

## Status

**Pre-engine, exploratory.** This is a tech-art exploration, not yet a committed
build. The engine choice (Bevy vs. Godot) is intentionally open — it depends on
whether the *tech* or the *art* turns out to be load-bearing for the early work.
Nothing here commits to either yet. The first concrete milestone is a **vertical
slice that stresses the hard seam** (cognitive level-of-detail), not a roadmap.
