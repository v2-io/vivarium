## Standing correction (Joseph 2026-07-23)

**Live code is not a co-equal truth source for architecture claims.** In this phase code is often the least principled layer. Rank when adjudicating promotions:

1. **Claim segments** (if already owned — do not dual-home)
2. **Design / theory / LEXICON / ordinum** as source material for what the project means
3. **DECISIONS** with honest `:by` / `:status` (authority ≠ evidence — still verify)
4. **Probes / measurements that can fail** for *behavior* claims
5. **Live code** last — as compliance debt, miniature, or falsifying instrument, never as "if unbuilt then not a claim"

"Not implemented" ⇒ compliance debt or residual build, **not** automatic reject of a formulation. "Implemented messily" ⇒ may still need the clean claim.
